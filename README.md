# ACMSI Data Competition Team Ajou Code

아주대학교 팀 **아주수학신세계트리오**의 ACMSI 데이터 경진대회 제출 코드입니다.

주제는 **지구 온난화에 따른 해수 온도, 빙하량, 해수 밀도 변화와 빙하기 도래 가능성**을 PINN(Physics-Informed Neural Network)으로 시뮬레이션하는 것입니다.

## 프로젝트 개요

이 프로젝트는 지구를 `60 x 100` 격자 좌표계로 단순화하고, 각 좌표의 시간 변화에 따라 다음 세 변수를 예측합니다.

- `T`: 온도
- `B`: 빙하량
- `rho`: 해수 밀도

모델은 입력 `(t, x, y)`를 받아 출력 `(T, B, rho)`를 예측하는 MLP 기반 PINN입니다. 관측 데이터에 맞추는 손실과 직접 설계한 물리 방정식의 잔차 손실을 함께 최소화하여, 관측 시점 이후의 변화를 추정합니다.

## 사용한 접근법

### 1. 데이터 기반 관측점 구성

두 개의 노트북은 같은 PINN 구조를 사용하지만 입력 데이터가 다릅니다.

| 노트북 | 목적 | 데이터 |
| --- | --- | --- |
| `아주수학신세계트리오_가상데이터_제출용.ipynb` | 가정 기반 시뮬레이션 | 세계 지도 이미지(`지구대륙.jpg`)에서 대륙/해양을 구분하고, 온난화가 진행되는 `t=0,1,2` 가상 관측 데이터를 생성 |
| `아주수학신세계트리오_실제데이터_제출용.ipynb` | 실제 데이터 기반 시뮬레이션 | Copernicus Marine 자료를 전처리한 2022-2024년 온도, 빙하량, 밀도 CSV 사용 |

실제 데이터 노트북에서는 원본 자료가 매우 커서 이미지화 및 리사이즈를 통해 `60 x 100` 크기로 축소한 CSV를 사용합니다. 대륙 지역의 누락 온도값은 Berkeley Earth Land/Ocean Temperature Record를 참고하여 보정한 것으로 코드 주석에 설명되어 있습니다.

### 2. PINN 모델

공통 모델 구조는 다음과 같습니다.

- 입력: `(t, x, y)`
- 출력: `(T, B, rho)`
- 네트워크: `MLP([3, 64, 64, 3])`
- 활성화 함수: `tanh`
- 옵티마이저: `optax.adam(1e-3)`
- 학습 반복: `20,000` epoch
- 손실 함수: `10000 * data_loss + 1 * physics_loss`

### 3. 물리 방정식

노트북에서 사용한 방정식은 다음 형태입니다.

```text
T_t = D * laplacian(T) + C(x, y, t) * T + G(t)
C(x, y, t) = C0 - k3 * abs(rho - rho_0)

B_t = -k1 * (T - Tm)
rho_t = k2 * B_t
```

여기서 `G(t)`는 온난화와 태양복사 에너지 증가를 반영하는 외부 forcing 항으로 사용됩니다.

PINN은 자동미분으로 `T_t`, `B_t`, `rho_t`, `laplacian(T)`를 계산하고, 위 방정식의 잔차가 작아지도록 학습합니다.

## 노트북별 설명

### 가상 데이터 노트북

`아주수학신세계트리오_가상데이터_제출용.ipynb`는 세계 지도 이미지에서 대륙과 해양을 구분한 뒤, 위도에 따른 온도·빙하량·해수 밀도 초기장을 직접 만듭니다.

주요 흐름은 다음과 같습니다.

1. `지구대륙.jpg`를 `60 x 100` 격자로 리사이즈
2. 이미지 thresholding으로 대륙과 해양 좌표 분리
3. `t=0`, `t=1`, `t=2`의 가상 온난화 관측 데이터 생성
4. 해양 좌표에서 collocation point `60,000`개 생성
5. PINN 학습 후 `t=0,2,5,10,20,30,50,80` 시점 시각화
6. 온도와 빙하량 변화를 mp4 애니메이션으로 저장 가능

가상 데이터의 PDE 계수는 `D=0.6`, `C0=0.6`, `k3=20.0`, `k1=0.5`, `k2=0.5`, `external_force=0.01`로 설정되어 있습니다.

### 실제 데이터 노트북

`아주수학신세계트리오_실제데이터_제출용.ipynb`는 전처리된 CSV 파일을 사용합니다.

주요 흐름은 다음과 같습니다.

1. 2022-2024년 온도, 빙하량, 해수 밀도 CSV 로드
2. 배열을 뒤집어 지도 좌표계와 맞추고 JAX 배열로 변환
3. 변수별 스케일 차이를 줄이기 위해 `0-6` 범위로 정규화
4. 모든 격자 좌표에서 collocation point `60,000`개 생성
5. PINN 학습 후 `t=0,2,5,10,15,30,50,80` 시점 시각화
6. 온도와 빙하량 변화를 mp4 애니메이션으로 저장 가능

실제 데이터의 PDE 계수는 `D=0.7`, `C0=0.7`, `k3=0.6`, `k1=0.5`, `k2=0.5`, `external_force=1.0`으로 설정되어 있습니다.

## 실행 방법

Python 3.11 환경에서 실행하는 것을 기준으로 작성되었습니다.

```bash
pip install jax jaxlib optax numpy pandas scipy pillow matplotlib notebook
```

애니메이션 저장까지 실행하려면 `ffmpeg`가 추가로 필요합니다.

```bash
jupyter notebook
```

노트북을 연 뒤 위에서 아래 순서대로 셀을 실행하면 됩니다. 학습 루프가 `20,000` epoch로 설정되어 있어 실행 환경에 따라 시간이 오래 걸릴 수 있습니다. mp4 저장 셀은 기본적으로 주석 처리되어 있으므로, 결과 영상을 새로 만들려면 마지막 애니메이션 저장 셀의 주석을 해제해야 합니다.
